#### **File: UpperServer.java**
```java
// Import the generated module and standard CORBA libraries
import UpperModule.*;
import org.omg.CORBA.*;
import org.omg.PortableServer.*;
import org.omg.CosNaming.*;

// This class starts the server and waits for client requests
class UpperServer {

    // The main entry point where the server execution starts
    public static void main(String args[]) {

        // Try-catch block helps catch and report network errors
        try {

            // Initialize the Object Request Broker (ORB) which handles messaging
            ORB orb = ORB.init(args, null);

            // Get a reference to the RootPOA which manages server objects
            POA rootpoa = POAHelper.narrow(
                    orb.resolve_initial_references("RootPOA"));

            // Tell the manager to start listening for requests
            rootpoa.the_POAManager().activate();

            // Create an instance of our logic class (UpperImpl)
            UpperImpl obj = new UpperImpl();

            // Register this object with the POA to make it a CORBA servant
            org.omg.CORBA.Object ref =
                    rootpoa.servant_to_reference(obj);

            // Convert the generic reference into the 'Upper' specific type
            Upper href = UpperHelper.narrow(ref);

            // Find the NameService, which is like a phonebook for services
            org.omg.CORBA.Object objRef =
                    orb.resolve_initial_references("NameService");

            // Prepare the phonebook tool to find or store service names
            NamingContextExt ncRef =
                    NamingContextExtHelper.narrow(objRef);

            // Give our service the name "Upper" so clients can find it
            ncRef.rebind(ncRef.to_name("Upper"), href);

            // Print a message to the console so we know the server is ready
            System.out.println("Server ready...");

            // Keep the server running and waiting for incoming client calls
            orb.run();

        // If something goes wrong, print the error message
        } catch (Exception e) {

            System.out.println("Error: " + e);
        // This brace ends the try-catch block
        }
    // This brace ends the main method
    }
// This brace ends the UpperServer class
}
```

### **2. STEP-BY-STEP EXECUTION GUIDE (For Linux Lab Machines)**


#### **Preparation**
Save the code into four files in a single folder: `Upper.idl`, `UpperImpl.java`, `UpperServer.java`, and `UpperClient.java`.

#### **Option 1: Single Machine Setup**
1.  **Generate CORBA files:** Open a terminal and type:
    `idlj -fall Upper.idl`
2.  **Compile all Java files:**
    `javac *.java UpperModule/*.java`
3.  **Start the Naming Service (The Broker):**
    `orbd -ORBInitialPort 1050 &`
4.  **Run the Server:**
    `java UpperServer -ORBInitialPort 1050 -ORBInitialHost localhost`
    *(Wait for "Server ready..." message)*
5.  **Run the Client (In a new terminal tab):**
    `java UpperClient -ORBInitialPort 1050 -ORBInitialHost localhost`
6.  **Expected Output:** The client asks for a string. If you type `hello`, it prints `Uppercase string: HELLO`.

Assume **Machine 1** is the Server and **Machine 2** is the Client.

**Step 1: Setup Machine 1 (Server)**

    `idlj -fall Upper.idl`

### **3. IMPORTANT THINGS TO TAKE CARE OF ON LINUX**

* **File Names:** Java is case-sensitive. `UpperServer.java` must match the class name `UpperServer` exactly.
* **Checking Tools:** Type `idlj -version` or `java -version` to check if Java is installed. If not, use `sudo apt install default-jdk`.
* **IP Address:** Use `hostname -I` or `ifconfig` in the terminal to find your Linux IP address.

* **Permissions:** If you can't run a file, use `chmod +x <filename>` to give it execution permissions.

### **4. CONCEPTS AND THEORY (For Viva)**

**CORBA (Common Object Request Broker Architecture)**
* **What it is:** A middleman technology that allows different computers to talk to each other.
* **Purpose:** To make remote objects look and act like local objects.
* **In this practical:** It provides the environment for `UpperClient` to call the `to_uppercase` method sitting on `UpperServer`.

**IDL (Interface Definition Language)**
* **What it is:** A standard language used to define how a client and server will talk without worrying about the programming language used.
* **Purpose:** It ensures that a C++ client could theoretically talk to this Java server.
* **In this practical:** `Upper.idl` defines the `to_uppercase` function name and its data types.

**ORB (Object Request Broker)**
* **What it is:** The communication backbone.
* **Purpose:** It finds the remote object, delivers the request, and brings back the result.
* [cite_start]**In this practical:** It is initialized in both Client and Server to handle the network traffic[cite: 1].

**Object Brokering (Naming Service)**
* **What it is:** A registry service (like a phone directory).
* **Purpose:** Servers "bind" their name in the directory so clients can "resolve" (find) them without knowing their exact location.
* [cite_start]**In this practical:** The Server binds the name "Upper," and the Client looks up "Upper" to find the server[cite: 1].

**Stub and Skeleton**
* **What they are:** Automatically generated helper code. The **Stub** stays with the Client; the **Skeleton** stays with the Server.
* **Purpose:** They hide the complexity of the network. The client thinks it's calling a local function; the stub handles the actual network transmission.
* **In this practical:** These are found inside the `UpperModule` folder generated by `idlj`.

**Client-Server Architecture**
* **What it is:** A model where one program (Client) requests a service and another (Server) provides it.
* [cite_start]**In this practical:** `UpperClient` requests a conversion, and `UpperServer` performs it[cite: 1].

**Which distributive technology is this?**
This is CORBA. Unlike RMI (which is only for Java), CORBA is designed to be language-independent. It uses an **IDL** and an **ORB** to manage communication.
